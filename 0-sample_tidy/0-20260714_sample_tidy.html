Pull real pepseq sample names

data_for_sharing_20250214 is the canonical Berry R21 share-out (see project notes on why this supersedes the undated data_for_sharing/). We read just the header row of each matrix (n_max = 0) rather than the full ~60MB file, since only the sample names are needed here.

igg_path <- "../pepsirf_data/data_for_sharing_20250214/IM0166_IM0174_PF1_raw_4mm_IgG_qc_Z-HDI85.tsv"
iga_path <- "../pepsirf_data/data_for_sharing_20250214/IM0166_PF1_raw_4mm_IgA_qc_Z-HDI85.tsv"

igg_samples <- colnames(read_tsv(igg_path, n_max = 0, show_col_types = FALSE))
iga_samples <- colnames(read_tsv(iga_path, n_max = 0, show_col_types = FALSE))

pepseq_sn <- tibble(pepseq_sn = c(igg_samples, iga_samples)) %>%
  filter(pepseq_sn != "Sequence name") %>%
  filter(str_detect(pepseq_sn, "Sblk", negate = TRUE)) %>%
  distinct() %>%
  mutate(isotype = case_when(
    str_detect(pepseq_sn, "_IgA_") ~ "IgA",
    str_detect(pepseq_sn, "_ProG_") ~ "IgG",
    TRUE ~ NA_character_
  )) %>%
  mutate(subject_id = str_split_fixed(pepseq_sn, "_", 3)[, 1]) %>%
  mutate(timepoint_or_date = str_split_fixed(pepseq_sn, "_", 3)[, 2]) %>%
  # ICS-format subject IDs are Mfera; plain numeric IDs are Liwonde
  # (same convention as the original 0-sample_tidy_v2.R)
  mutate(cohort = case_when(
    str_detect(subject_id, "ICS") ~ "Mfera",
    TRUE ~ "Liwonde"
  )) %>%
  # Some draws were sequenced on more than one pepsirf run (e.g. subject 179's
  # 16-Mar-18 IgG draw exists under both IM0174 and IM0163) -- keep the run ID
  # so downstream joins/pivots don't silently collapse genuinely distinct
  # samples that share the same (subject_id, timepoint_or_date, isotype).
  mutate(run_id = str_extract(pepseq_sn, "IM\\d{4}"))

print(paste0("Total samples: ", nrow(pepseq_sn), " (IgG: ", sum(pepseq_sn$isotype == "IgG"),
             ", IgA: ", sum(pepseq_sn$isotype == "IgA"), ")"))

Mfera cohort

The manifest’s “Mfera” sheet gives the exact (ICS subject_id, Infant Month code) pair per real tube, which matches the second token of the real pepseq sample names directly – no reconstruction needed. mfera_share_20aug2024.csv only has a plain integer mfera_id; the 6-digit segment embedded in the ICS code (ICS-000020-2 -> 000020) is that integer, confirmed against the CSV during planning.

mfera_share_20aug2024.csv has one row per infection episode (like Liwonde), each naming its serum1/serum2 draws via serum1e/serum2e tube codes (e.g. 020-M18 = mfera_id 020 + Infant Month code M18). Joining the manifest to this CSV on mfera_id alone fans out (a subject with 2+ episodes has 2+ clinical rows, but the manifest only has one row per tube) – so, same fix as Liwonde: pivot the clinical data long per draw and join on (mfera_id, timepoint_or_date) for a precise 1:1 match.

Liwonde cohort

Real sample names encode the collection date directly (e.g. 179_16-Mar-18 or 508_16-Nov-2018 – note the inconsistent 2- vs 4-digit year, which is exactly what broke the original string-reconstruction approach in 0-sample_tidy_v2.R). Instead of rebuilding a name to match, we parse the real dates and match them against clinical serum1/serum2 dates – comparing real data to real data.

Combine cohorts + plate assignment

PF1_plate_well_sequencingrun.xlsx (from Alexis, filling the gap in her IgA batch-effect analysis) has SampleName values that match our real pepseq sample names exactly, so this is a plain string join – no reconstruction. This preserves plate assignment in the tidy output; it does not correct the batch effect itself (Alexis found raw IgA counts from run IM0163 cluster almost perfectly by Plate 2 vs Plate 4, persisting even in Z-scores – that correction, e.g. via ComBat, is a downstream analysis decision, out of scope here).

That file’s IM0163 coverage turned out to be a partial export (only ~half of Mfera subjects). IM0163_plate_info.xlsx is the full wet-lab tracking workbook for that run; its NEW ZZSHEET-PF1 tab gives complete IM0163 coverage but in a 4-plates-side-by-side print layout rather than a flat list – 4 repeated column blocks of Index1 | Index2 | SampleName | [blank] | plate | well, starting at columns 1, 8, 15, 22. Reassemble by reading blocks separately and stacking them.

Not fully resolved: this tab’s plate numbers (1-12, spanning both isotypes per plate) don’t obviously correspond to the workbook’s separate “Plate 1”-“Plate 7” tab names elsewhere in the same file – worth a sanity check with whoever maintains this workbook before treating the numbering as more than “which physical batch”.

extract_plate_block <- function(df, start_col) {
  df %>%
    select(all_of(start_col:(start_col + 5))) %>%
    set_names(c("Index1", "Index2", "pepseq_sn", "blank_sep", "plate_id", "well")) %>%
    select(-blank_sep) %>%
    filter(!is.na(pepseq_sn))
}

plate_map_im0163 <- bind_rows(
  extract_plate_block(zzsheet_pf1_raw, 1),
  extract_plate_block(zzsheet_pf1_raw, 8),
  extract_plate_block(zzsheet_pf1_raw, 15),
  extract_plate_block(zzsheet_pf1_raw, 22)
) %>%
  select(pepseq_sn, plate_id, well) %>%
  # normalize "1.0" -> "Plate 1" to match PF1_plate_well_sequencingrun.xlsx's format
  mutate(plate_id = paste0("Plate ", as.integer(as.numeric(plate_id))))

plate_map <- bind_rows(plate_map_im0163, plate_map_im0174) %>%
  distinct(pepseq_sn, .keep_all = TRUE)

  left_join(plate_map, by = "pepseq_sn")

Pairs

Berry R21’s design pairs a pre/post sample per infection episode (serum1/serum2), unlike msp1’s sequential multi-timepoint design – so pairs come directly from the clinical join rather than a generic timepoint-sequence helper.

Both cohorts now share the same record_id/serum_which/pepseq_sn shape after tidying (Mfera’s serum_which values are serum1e/serum2e, Liwonde’s are serum1/serum2), so pairs can be built the same way for both.

Each clinical draw has two real pepseq samples (one IgG, one IgA), and a handful of draws were sequenced on more than one run (e.g. subject 179’s 16-Mar-18 IgG draw exists under both IM0174 and IM0163) – isotype and run_id both have to stay in the grouping key alongside subject_id/record_id, otherwise pivot_wider can’t resolve a single pepseq_sn per cell.

Show code 
pairs_liwonde <- liwonde_tidy %>%
  filter(!is.na(serum_which)) %>%
  select(subject_id, record_id, isotype, run_id, serum_which, pepseq_sn) %>%
  pivot_wider(names_from = serum_which, values_from = pepseq_sn) %>%
  filter(!is.na(serum1), !is.na(serum2))

pairs_mfera <- mfera_tidy %>%
  filter(!is.na(serum_which)) %>%
  select(subject_id, record_id, isotype, run_id, serum_which, pepseq_sn) %>%
  pivot_wider(names_from = serum_which, values_from = pepseq_sn) %>%
  filter(!is.na(serum1e), !is.na(serum2e))

```{r}
  print("Duplicate (subject_id, date) pairs in Liwonde clinical data -- excluded from the join below, written out for manual review:")
  print(dup_liwonde_dates)
}

# Same approach as Mfera above: exclude ambiguous records from the join so
# it stays 1:1, instead of silently fanning out.
meta_liwonde_long_clean <- meta_liwonde_long %>%
  anti_join(dup_liwonde_dates, by = c("subject_id", "parsed_date"))

liwonde_tidy <- liwonde_pepseq %>%
  left_join(meta_liwonde_long_clean, by = c("subject_id", "parsed_date"))

# QC: clinical records with no matching real pepseq sample
liwonde_missing_pepseq <- meta_liwonde_long %>%
  select(subject_id, parsed_date, record_id, serum_which) %>%
  anti_join(liwonde_pepseq, by = c("subject_id", "parsed_date"))

# QC: real pepseq samples with no matching clinical record
liwonde_missing_clinical <- liwonde_pepseq %>%
  distinct(subject_id, parsed_date) %>%
  anti_join(meta_liwonde_long, by = c("subject_id", "parsed_date"))

print(paste0("Liwonde: ", nrow(liwonde_missing_pepseq), " clinical records missing PepSeq data, ",
             nrow(liwonde_missing_clinical), " PepSeq samples missing from clinical data"))
```

## Combine cohorts + plate assignment

`PF1_plate_well_sequencingrun.xlsx` (from Alexis, filling the gap in her IgA
batch-effect analysis) has `SampleName` values that match our real pepseq sample names
exactly, so this is a plain string join -- no reconstruction. This *preserves* plate
assignment in the tidy output; it does not correct the batch effect itself (Alexis
found raw IgA counts from run IM0163 cluster almost perfectly by Plate 2 vs Plate 4,
persisting even in Z-scores -- that correction, e.g. via ComBat, is a downstream
analysis decision, out of scope here).

That file's IM0163 coverage turned out to be a partial export (only ~half of Mfera
subjects). `IM0163_plate_info.xlsx` is the full wet-lab tracking workbook for that run;
its `NEW ZZSHEET-PF1` tab gives complete IM0163 coverage but in a 4-plates-side-by-side
print layout rather than a flat list -- 4 repeated column blocks of
`Index1 | Index2 | SampleName | [blank] | plate | well`, starting at columns 1, 8, 15,
22. Reassemble by reading blocks separately and stacking them.

Not fully resolved: this tab's plate numbers (1-12, spanning both isotypes per plate)
don't obviously correspond to the workbook's separate "Plate 1"-"Plate 7" tab names
elsewhere in the same file -- worth a sanity check with whoever maintains this workbook
before treating the numbering as more than "which physical batch".

```{r}
plate_map_im0174 <- read_excel("../sample_data/Shared sample information/PF1_plate_well_sequencingrun.xlsx") %>%
  rename(pepseq_sn = SampleName) %>%
  select(pepseq_sn, plate_id, well) %>%
  mutate(plate_id = as.character(plate_id))

zzsheet_pf1_raw <- suppressMessages(read_excel(
  "../sample_data/Shared sample information/IM0163_plate_info.xlsx",
  sheet = "NEW ZZSHEET-PF1", col_names = FALSE, skip = 2
))

extract_plate_block <- function(df, start_col) {
  df %>%
    select(all_of(start_col:(start_col + 5))) %>%
    set_names(c("Index1", "Index2", "pepseq_sn", "blank_sep", "plate_id", "well")) %>%
    select(-blank_sep) %>%
    filter(!is.na(pepseq_sn))
}

plate_map_im0163 <- bind_rows(
  extract_plate_block(zzsheet_pf1_raw, 1),
  extract_plate_block(zzsheet_pf1_raw, 8),
  extract_plate_block(zzsheet_pf1_raw, 15),
  extract_plate_block(zzsheet_pf1_raw, 22)
) %>%
  select(pepseq_sn, plate_id, well) %>%
  # normalize "1.0" -> "Plate 1" to match PF1_plate_well_sequencingrun.xlsx's format
  mutate(plate_id = paste0("Plate ", as.integer(as.numeric(plate_id))))

plate_map <- bind_rows(plate_map_im0163, plate_map_im0174) %>%
  distinct(pepseq_sn, .keep_all = TRUE)

sample_tidy_all <- bind_rows(mfera_tidy, liwonde_tidy) %>%
  left_join(plate_map, by = "pepseq_sn")

print(paste0(sum(!is.na(sample_tidy_all$plate_id)), " of ", nrow(sample_tidy_all),
             " samples matched to a plate assignment"))
```

## Pairs

Berry R21's design pairs a pre/post sample per infection episode (`serum1`/`serum2`),
unlike msp1's sequential multi-timepoint design -- so pairs come directly from the
clinical join rather than a generic timepoint-sequence helper.

Both cohorts now share the same `record_id`/`serum_which`/`pepseq_sn` shape after
tidying (Mfera's `serum_which` values are `serum1e`/`serum2e`, Liwonde's are
`serum1`/`serum2`), so pairs can be built the same way for both.

Each clinical draw has *two* real pepseq samples (one IgG, one IgA), and a handful of
draws were sequenced on more than one run (e.g. subject 179's `16-Mar-18` IgG draw
exists under both `IM0174` and `IM0163`) -- `isotype` and `run_id` both have to stay in
the grouping key alongside `subject_id`/`record_id`, otherwise `pivot_wider` can't
resolve a single `pepseq_sn` per cell.

```{r}
pairs_liwonde <- liwonde_tidy %>%
  filter(!is.na(serum_which)) %>%
  select(subject_id, record_id, isotype, run_id, serum_which, pepseq_sn) %>%
  pivot_wider(names_from = serum_which, values_from = pepseq_sn) %>%
  filter(!is.na(serum1), !is.na(serum2))

pairs_mfera <- mfera_tidy %>%
  filter(!is.na(serum_which)) %>%
  select(subject_id, record_id, isotype, run_id, serum_which, pepseq_sn) %>%
  pivot_wider(names_from = serum_which, values_from = pepseq_sn) %>%
  filter(!is.na(serum1e), !is.na(serum2e))

# Diagnostics matching msp1's style
print(paste0("Liwonde subjects with complete serum1/serum2 pairs: ", nrow(pairs_liwonde)))
print(paste0("Mfera subjects with complete serum1e/serum2e pairs: ", nrow(pairs_mfera)))
```
